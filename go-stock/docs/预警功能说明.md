# AI 推荐股票预警功能说明

## 功能概述

AI 推荐股票预警功能是一个智能化的股价监控通知系统，当 AI 推荐股票的价格达到预设的预警线时，系统会自动发送通知到 Windows 系统和钉钉群，帮助投资者及时把握买卖时机。

## 功能特点

1. **多类型预警**：支持买入预警、止盈预警、止损预警三种类型
2. **多渠道通知**：支持 Windows 系统通知和钉钉群通知
3. **预警频率控制**：同一预警类型 5 分钟内不会重复发送
4. **灵活开关控制**：每条推荐记录可独立开启/关闭预警
5. **智能筛选查询**：支持按预警状态筛选推荐记录

## 预警类型

### 1. 买入预警
- **触发条件**：当前价格 ≤ AI 建议买入价最低价
- **通知内容**：股票名称、股票代码、当前价格、建议买入价范围、推荐时间
- **使用场景**：当股价下跌到 AI 建议的买入价格区间时提醒入场

### 2. 止盈预警
- **触发条件**：当前价格 ≥ AI 建议止盈价最低价
- **通知内容**：股票名称、股票代码、当前价格、建议止盈价范围、推荐时间
- **使用场景**：当股价上涨到预期收益目标时提醒获利了结

### 3. 止损预警
- **触发条件**：当前价格 ≤ AI 建议止损价
- **通知内容**：股票名称、股票代码、当前价格、建议止损价、推荐时间
- **使用场景**：当股价下跌到风险控制位置时提醒及时止损

## 使用指南

### 前端操作

#### 1. 开启/关闭预警
在「股票推荐记录」页面，每条记录都有独立的预警开关（NSwitch 组件）：

- 开启预警：点击开关，变为绿色状态
- 关闭预警：点击开关，变为灰色状态

#### 2. 按预警状态筛选
在搜索栏中有「预警状态」下拉框，支持以下筛选选项：

- **全部**：显示所有推荐记录
- **已开启预警**：只显示开启预警的记录
- **未开启预警**：只显示未开启预警的记录

#### 3. 查看详情
点击记录操作列中的「查看」按钮，可以查看该股票的详细分析和建议。

### 后台配置

#### 钉钉通知配置
1. 打开应用设置
2. 找到「钉钉推送」选项
3. 开启钉钉推送
4. 填写钉钉机器人 Webhook 地址

## 技术实现

### 数据模型

```go
type AiRecommendStocks struct {
    // ... 其他字段
    RecommendBuyPriceMin        float64  // AI 建议最低买入价
    RecommendBuyPriceMax        float64  // AI 建议最高买入价
    RecommendStopProfitPriceMin float64  // AI 建议最低止盈价
    RecommendStopProfitPriceMax float64  // AI 建议最高止盈价
    RecommendStopLossPrice      float64  // AI 建议止损价
    EnableAlert                 bool     // 是否开启预警
}
```

### 监控频率
- 系统每 60 秒检查一次开启预警的股票价格
- 仅在 A股、港股、美股开市时执行监控

### 通知格式
通知内容使用 Markdown 格式，便于在钉钉中清晰展示：

```
> **股票名称**: XXX
> **股票代码**: XXXXXX
> **当前价格**: XX.XX
> **建议买入价**: XX.XX - XX.XX
> **推荐时间**: 2026-03-23 10:30:00
```

### 预警冷却机制
为避免频繁通知打扰，同一股票的同一预警类型设置了 5 分钟的冷却时间：

```
预警标识 = 股票代码:预警类型:推荐日期
示例：000001:BUY:20260323
```

## 数据库字段

如需手动添加数据库字段，执行以下 SQL：

```sql
ALTER TABLE ai_recommend_stocks ADD COLUMN enable_alert BOOLEAN DEFAULT FALSE;
```

## 注意事项

1. **预警只是提醒**：预警通知仅供参考，不构成投资建议
2. **市场风险**：股市有风险，投资需谨慎，请根据实际情况判断
3. **网络畅通**：钉钉通知需要网络畅通才能正常接收
4. **数据库迁移**：首次使用需要确保数据库已添加 `enable_alert` 字段

## 故障排查

### 预警通知不发送
1. 检查钉钉推送是否已开启
2. 检查钉钉机器人 Webhook 地址是否正确
3. 检查系统通知权限是否授权
4. 检查是否开启了对应记录的预警开关

### 预警状态筛选不生效
1. 检查数据库是否已添加 `enable_alert` 字段
2. 重启应用以加载最新的代码逻辑

### 预警价格不准确
1. 检查 AI 推荐记录的建议价格是否正确
2. 确认使用的是最新的推荐数据
