# 2026-03-27 AkShare 中国市场补充层方案

## 结论

AkShare 适合接入当前系统，但只应作为“中国市场补充层”，不应接管主链。

原因不是它不够强，而是当前系统更需要：

1. 保持 `market-mcp -> latest.json -> risk_dashboard.json -> briefs` 这一条主链稳定
2. 把 AkShare 用在它最强的中国市场长尾数据上
3. 避免双数据口径同时污染主行情、主估值和主风控

## 本次落地位置

本次新增：

1. `scripts/generate_cn_market_snapshot.py`
2. `cn_market_snapshots/YYYY-MM-DD-cn-snapshot.json`
3. `scripts/lib/cn_market_snapshot.mjs`

当前接入方式是“旁路补充”：

1. 先独立生成中国市场补充快照
2. 再由 `generate_market_brief.mjs` 和 `generate_daily_brief.mjs` 按需读取
3. 若快照不存在或依赖缺失，主日报继续按原链路生成

## 当前已接的 5 类字段

### 1. A 股市场宽度

用于看：

1. 上涨/下跌家数
2. 中位涨跌幅
3. 成交额前排个股

这类信息适合回答“今天 A 股内部到底是普涨、普跌，还是少数权重拉着走”。

### 2. 北向资金

用于看：

1. 通道状态
2. 当日净买额或盘中净流入
3. 若当前端点回零，则自动降级为“只做辅助核验”

这类信息很适合给 A 股修复与板块轮动做补充核验。

### 3. 国内宏观相位

用于看：

1. 制造业 PMI
2. CPI 同比
3. M2 同比
4. 当前更偏修复还是偏收缩

这类信息适合给高股息、宽基核心、制造升级这些风格判断提供背景板。

### 4. 板块资金流

用于看：

1. 行业资金流今日前三
2. 行业资金流 5 日前三
3. 概念资金流今日前三
4. 资金正在主动回流哪些方向、主动流出哪些方向

这类信息适合回答“今天的强势到底只是价格涨了，还是主力资金也真的在往里打”。

### 5. sector rotation 验证

用于看：

1. 今日主线风格
2. 近 5 日主线风格
3. 是趋势延续、快速轮动、还是防守漂移
4. 当前确认度高、中、还是低

这类信息适合把“板块很强”进一步变成“这是不是一个有延续性的结构信号”。

## 明确不接管的部分

AkShare 当前不应直接接管：

1. `latest.json`
2. `risk_dashboard.json`
3. 基金估值主链
4. 港股、黄金、CME、全球风险链条

这些仍由现有 `market-mcp` 主链负责。

## 为什么这才像“补充层”

因为它只负责回答以下问题：

1. 今天 A 股内部宽度到底强不强
2. 北向资金有没有共振
3. 国内宏观背景更支持防守还是进攻
4. 今天的板块强势有没有资金确认
5. 当前轮动是不是在延续，还是只是短线切换

而不是直接回答：

1. 组合净值是多少
2. 明天基金一定该买什么
3. 跨市场风险是不是已经解除
