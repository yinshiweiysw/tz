# AI 辅助交易系统蓝图

更新时间：2026-03-25

## 目标

通过 Codex + 持仓档案 + 市场数据 + 规则系统，形成一套长期可续跑、可复盘、可验证的 AI 辅助交易系统。

这套系统的目标不是替代判断，而是：

1. 降低情绪化交易
2. 提高规则执行率
3. 保留利润，控制回撤
4. 让交易结论可以长期沉淀和复用

## 当前已完成

### 1. 状态与续跑

- `snapshots/latest_raw.json` 作为原始平台快照
- `ledger/execution_ledger.json` 作为执行流水
- `state/portfolio_state.json` 作为当前策略主状态
- `latest.json` 作为兼容物化视图
- `state-manifest.json` 作为无记忆续跑入口
- `OPERATING_PROTOCOL.md` 作为更新规则

### 2. 日常沉淀

- `journal/daily/YYYY-MM-DD.md` 作为每日纪要
- `daily_writeback.mjs` 作为关键聊天/交易写回工具
- `transactions/` 作为手工交易流水
- `reports/` 作为每日分析与专项报告

### 3. 周度复盘

- `SCORING_RUBRIC.md` 作为评分标准
- `scorecards/weekly/YYYY-Www-scorecard.md` 作为每周评分文件

### 4. 市场与基金数据

- `market-mcp` 已整合 `go-stock` 和 `funds`
- 已支持股票分析、基金估值、观察名单净值查询
- 已支持生成独立的市场日报 `market_briefs/YYYY-MM-DD-market.md`

### 5. 双引擎交易架构

当前主链已经收敛为三层：

- `L1 研究发现层`
  负责主题研究、机会池与预期差扫描，核心产物是 `data/opportunity_pool.json`
- `L2 信号与约束层`
  负责动量、估值、宏观、相关性与风险矩阵，核心输入包括 `signals_matrix.json`、`risk_dashboard.json`、`macro_radar.json`、`quant_metrics_engine.json`
- `L3 执行层`
  负责把主系统计划与左侧博弈计划并列渲染为双轨交易预案，核心产物是 `reports/*-next-trade-plan-regime-v4.md`

两套引擎的边界如下：

- `Core Engine`
  负责 IPS 骨架、仓位修复、再平衡、现金防线与风险约束
- `Speculative Engine`
  负责受限预算下的左侧试错，只能表达小额博弈，不承担骨架修复职责

这意味着当前系统已经不再是“一个建议器”，而是“研究发现 -> 风险约束 -> 双轨执行”的串行架构。

## 还需要补的核心模块

### A. 投资政策声明（必须优先）

建议新增一个长期主文件：

- `INVESTMENT_POLICY_STATEMENT.md`

内容应明确：

- 总目标：增值、稳健、还是兼顾弹性
- 最大可承受回撤
- 常态目标仓位结构
- 哪些资产是核心仓
- 哪些资产只能做交易仓
- 什么情况下一定减仓
- 什么情况下一定暂停交易

意义：

- 这是全系统最高优先级规则文件
- 后续所有日常建议都应优先服从它

### B. 交易决策树（必须优先）

建议新增：

- `DECISION_TREE.md`

覆盖情景：

- 高开
- 低开
- 冲高回落
- 跌深反弹
- 地缘缓和
- 地缘升级
- 踏空焦虑
- 回本冲动

意义：

- 把模糊判断变成条件判断
- 降低盘中上头概率

### C. 交易卡片体系（强烈建议）

建议新增目录：

- `trade_cards/`

每次重要交易都生成一张卡片，至少记录：

- 标的
- 买入/卖出理由
- 仓位大小
- 预期逻辑
- 失效条件
- 目标动作

意义：

- 避免“买的时候很有道理，过几天忘了为什么买”
- 后续能复盘是逻辑错了还是执行错了

当前状态：

- 已完成第一版目录、模板和生成脚本
- 已建立首张交易卡片作为样例
- 后续每次重要交易都应补卡，不再只保留流水

### D. 风险仪表盘（强烈建议）

建议新增一个自动汇总文件：

- `risk_dashboard.json`

至少包含：

- 当前总仓位
- 现金比例
- 单一主题占比
- 高相关资产占比
- 港股科技占比
- 黄金占比
- QDII 占比
- 本周累计交易次数

意义：

- 以后不用靠感觉判断自己是不是又重仓了
- 可以直接拿数据提醒“超限”

当前状态：

- 已完成第一版
- 已支持 `canonical_view` 与 `working_view`
- 后续可继续增加自动预警和更细的主题拆分

### E. 假设跟踪器（建议）

建议新增：

- `hypotheses.md`

用于记录类似：

- “中东局势缓和将推动港股科技修复”
- “黄金仍然处于强趋势”
- “A股红利承担稳定器”

每条假设都要有：

- 建立日期
- 支撑证据
- 反证条件
- 当前状态：进行中 / 已验证 / 已失效

意义：

- 避免把观点变成信仰
- 强迫系统承认“判断失效”

当前状态：

- 已完成第一版主文件和模板
- 已录入当前最重要的 5 条假设
- 后续应在每周评分时同步检查其状态

### F. 组合日报（建议）

建议未来每天自动生成：

- `daily_briefs/YYYY-MM-DD-brief.md`

固定包含：

- 当日持仓变化
- 估值变化
- 市场重点消息
- 是否触发规则
- 明日关键观察点

意义：

- 这是最适合长期连续阅读的轻量日报

当前状态：

- 已完成第一版目录、模板和脚本
- 已生成首份样例日报
- 后续每个交易日收口时都应生成

## 长期最重要的三条原则

### 1. 先做风险系统，再做选股系统

如果没有风险系统，任何观点都可能被仓位失控毁掉。

### 2. 先做记录系统，再做优化系统

没有连续记录，就无法真正知道自己是在进步，还是只是碰运气。

### 3. 先做可执行规则，再做复杂模型

你现在最需要的不是更复杂的预测，而是：

- 更稳定的规则
- 更一致的执行
- 更快的复盘闭环

## Codex 的定位

Codex 在这套系统里的角色不只是“回答问题”，而是：

- 你的交易记录员
- 你的风险监察员
- 你的复盘助理
- 你的规则执行提醒器
- 你的市场情报整理助手

但 Codex 不应该变成：

- 无条件替你拍板的黑箱
- 帮你放大情绪交易的理由制造器

## 一句话总结

你想要的不是一个“会聊天的投顾”，而是一套“能把你从情绪交易者慢慢改造成系统交易者”的 AI 协作框架。
