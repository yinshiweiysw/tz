# Trade Cards

这个目录用于保存每一笔重要交易的决策卡片。

## 作用

交易卡片不是流水，也不是复盘摘要，而是“这笔交易当时为什么成立”的结构化记录。

它主要解决三个问题：

1. 以后不会忘记当时为什么买/卖
2. 可以回头判断是逻辑错了还是执行错了
3. 能把“我感觉”变成“有失效条件的假设”

## 命名规范

- `trade_cards/YYYY/YYYY-MM-DD-*.md`

## 最低记录要求

- 标题
- 日期
- 交易方向
- 仓位角色
- 标的与金额
- 核心逻辑
- 失效条件
- 风险点
- 后续计划

## 与其他文件的关系

- `transactions/` 记录“发生了什么”
- `journal/daily/` 记录“当天重要结论”
- `trade_cards/` 记录“为什么这么做，以及什么时候算错”
